## 1. Idea inicial del agente

Queremos construir un asistente sencillo llamado:

```text
MiniJarvis Ada
```

En la primera versión no tendrá IA real.

Será un programa Java por consola que salude, pida el nombre y explique que acompañará el aprendizaje durante el curso.

---

## 2. Qué podría entrar en H1

| Tarea | Motivo |
|---|---|
| Crear proyecto Java en IntelliJ | Necesitamos una base para empezar. |
| Crear clase `Main` | Será el punto de entrada del programa. |
| Mostrar saludo inicial | Primer comportamiento visible. |
| Pedir nombre al usuario | Primera entrada por teclado. |
| Guardar nombre en variable | Practicar variables. |
| Crear constante con el nombre del asistente | Practicar constantes. |
| Mostrar mensajes del proyecto | Conectar con MiniJarvis. |
| Escribir README básico | Explicar cómo ejecutar. |
| Probar ejecución | Comprobar que funciona. |

---

## 3. Qué NO debería entrar todavía

| Idea | Por qué no entra en H1 |
|---|---|
| Menú de comandos | Se trabajará en H2. |
| Bucles | Se trabajarán en H2. |
| Memoria de mensajes | Se trabajará en H3. |
| Varias clases | Se trabajará en H4. |
| Conexión con Gemini/Jarvis | Es demasiado pronto y necesita seguridad. |
| Guardar datos en ficheros | Se trabajará más adelante. |

---

## 4. Primeros riesgos del equipo

| Riesgo | Cómo lo reduciremos |
|---|---|
| Querer hacer demasiadas cosas en H1 | Revisar la lista de cosas que no entran todavía. |
| Copiar código de IA sin entender | Registrar el uso y defender cada línea. |
| No saber ejecutar en IntelliJ | Pedir ayuda pronto y documentar pasos. |
| Repartir mal el trabajo | Usar backlog y roles. |

---

## 5. Qué significa que H1 esté terminado

Para nosotros, H1 estará terminado si:

```text
El programa se ejecuta.
Pide el nombre.
Responde usando ese nombre.
Tiene al menos una constante.
Tiene README.
Cada persona puede explicar las líneas principales.
```
