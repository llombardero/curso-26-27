- portfolio y defensa

## 1. Punto de partida

MiniJarvis se construye por capas. Este capítulo trabaja una pieza concreta del proyecto y la conecta con evidencias reales: código, pruebas, documentación, defensa y uso responsable de IA si procede.

La regla de estudio es siempre la misma:

```text
Lee el concepto, ejecuta un ejemplo pequeño, aplícalo a MiniJarvis, documenta la evidencia y prepárate para defenderlo.
```

## 2. Conceptos básicos explicados

El libro no sustituye a las clases ni a las prácticas: las acompaña. Cada capítulo se puede leer como una unidad de estudio independiente. La parte de Programación se centra en construir código Java progresivamente. La parte de Entornos se centra en documentar, probar, depurar, versionar, revisar y defender.

Un hito es una etapa con un producto parcial. Una evidencia es una prueba concreta de aprendizaje. Una defensa es la explicación oral o escrita que demuestra que comprendes lo entregado. Si una evidencia no se puede localizar, probar o explicar, todavía está incompleta.

MiniJarvis sirve como hilo conductor: no aprendes conceptos sueltos, sino conceptos aplicados a un producto que crece durante el curso.

## 4. Caso práctico MiniJarvis

Crea una carpeta de portfolio personal. Dentro, crea un archivo `indice-portfolio.md` con una tabla H0-H7-HF. Para cada hito deja preparado un hueco: evidencia, prueba, documento y defensa.

Caso práctico: escribe una primera entrada titulada “Qué espero aprender construyendo MiniJarvis”. Incluye una fortaleza, una dificultad prevista y una norma personal de uso de IA.

## 5. Evidencia de Entornos de Desarrollo

Crea un README inicial de tu carpeta personal. Debe explicar qué contiene, cómo se irán guardando evidencias y qué regla seguirás para registrar IA. Esto ya es Entornos: documentación, organización y trazabilidad.

## 6. Errores frecuentes y cómo corregirlos

| Error | Síntoma | Corrección |
|---|---|---|
| Leer capítulos sin practicar | Entiendes la teoría pero no puedes aplicarla | Ejecuta siempre el caso práctico |
| Guardar evidencias desordenadas | No encuentras nada al defender | Usa nombres claros y tabla de hitos |
| Usar IA sin registrar | No puedes justificar el proceso | Anota prompt, cambio y verificación |

## 7. Preguntas de repaso

2. ¿Por qué el portfolio no es solo una carpeta de archivos?

4. ¿Qué debes registrar si usas IA?

## Ajuste editorial — cobertura completa de conceptos de Programación

Este libro queda alineado con `32-lista-conceptos-programacion-por-tema.md`.

La cobertura completa se consigue así:

| Bloque del libro | Temas cubiertos |
