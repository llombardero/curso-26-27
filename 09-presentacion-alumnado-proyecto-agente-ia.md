08-guia-alumnado-proyecto-agente-ia.md

## 1. Bienvenida al proyecto

Este curso vamos a aprender Programación y Entornos de Desarrollo construyendo un proyecto común:

```text
MiniJarvis: un pequeño agente IA propio
```

No será un proyecto que aparezca terminado desde el primer día.

Lo construiremos poco a poco, por versiones, aprendiendo en cada fase algo nuevo:

- Java;
- IntelliJ;
- Git y GitHub;
- pruebas;
- depuración;
- documentación;
- diagramas;
- trabajo en equipo;
- código limpio;
- uso responsable de IA.

La meta no es que una IA haga el trabajo por nosotros.

La meta es aprender a construir software que podamos entender, probar, explicar y mejorar.

---

## 2. Qué construirás

Construirás, individualmente y en equipo, un asistente por consola en Java.

Al principio será muy sencillo.

Después irá creciendo:

| Momento | Qué aprenderemos |
|---|---|
| Inicio | Scrum, roles, trabajo en equipo y organización del proyecto. |
| H1 | Primer programa Java básico por consola. |
| H2 | Menú, comandos, decisiones, bucles y depuración. |
| H3 | Memoria temporal usando colecciones. |
| H4 | Clases, objetos y diseño orientado a objetos. |
| H5 | Herramientas internas, refactorización, Git profesional y código limpio. |
| H6 | Ficheros, logs, persistencia y ejecución reproducible. |
| H7 | Integración con IA real o simulación robusta, si el grupo está preparado. |
| Final | Demo, portfolio, defensa individual, recuperación y mejora. |

El proyecto podrá incluir IA real, como Gemini o Jarvis, solo cuando tenga sentido y se pueda hacer de forma segura.

---

## 3. Cómo trabajaremos

Trabajaremos por hitos.

Un hito es una versión parcial del proyecto con un objetivo concreto.

En cada hito habrá:

1. un reto;
2. tareas técnicas;
3. entregables;
4. pruebas o comprobaciones;
5. documentación;
6. uso controlado de IA, si procede;
7. defensa o revisión.

También usaremos Scrum adaptado al aula.

Esto significa que los equipos tendrán:

- roles;
- tablero de tareas;
- acuerdos de equipo;
- revisiones;
- retrospectivas;
- demos.

No se espera que todo salga perfecto a la primera.

Sí se espera que el equipo aprenda a organizarse, detectar errores, corregirlos y explicar su proceso.

---

## 4. Qué tendrás que entregar

Habrá entregas de equipo y entregas individuales.

### Entregas de equipo

El equipo podrá entregar:

- código Java;
- repositorio GitHub o entrega equivalente;
- README;
- pruebas;
- incidencias;
- diagramas;
- informes de depuración;
- decisiones técnicas;
- demo del proyecto.

### Entregas individuales

Cada alumno/a podrá entregar o defender:

- portfolio individual;
- registro de uso de IA;
- comparación Java ↔ Python;
- explicación de una parte del código;
- reflexión sobre lo aprendido;
- defensa oral;
- recuperación específica si hace falta.

Una entrega no consiste solo en subir archivos.

Una entrega válida debe permitir comprobar:

```text
Funciona.
Está documentada.
La puedes explicar.
```

---

## 5. Reglas de uso de IA

La IA se puede usar en este curso.

Pero se usará con responsabilidad.

Puedes usar IA para:

- entender conceptos;
- pedir ejemplos pequeños;
- revisar errores;
- preparar pruebas;
- mejorar documentación;
- comparar Java con Python;
- preparar una defensa;
- revisar tu portfolio o README.

No puedes usar IA para:

- copiar una solución completa sin entenderla;
- ocultar que la has usado;
- entregar código que no sabes explicar;
- usarla en pruebas individuales si no está autorizado;
- introducir contraseñas, claves, tokens o datos personales;
- subir secretos al repositorio.

Regla principal:

```text
La IA puede ayudarte a aprender.
La IA no puede aprender por ti.
```

Si usas IA en una entrega evaluable, tendrás que registrarlo en:

```text
docs/registro-ia.md
```

Y tendrás que poder explicar:

- qué pediste;
- qué te respondió;
- qué aceptaste;
- qué cambiaste tú;
- cómo lo comprobaste;
- qué aprendiste.

---

## 6. Cómo se evaluará

El proyecto servirá para evaluar evidencias de dos módulos:

- Programación;
- Entornos de Desarrollo.

Aunque el proyecto sea común, cada módulo se evaluará por separado.

Se valorará:

- que el código funcione;
- que el nivel sea adecuado al hito;
- que el código sea claro;
- que haya pruebas o comprobaciones;
- que el README explique cómo ejecutar;
- que el trabajo esté documentado;
- que haya trazabilidad en Git/GitHub cuando corresponda;
- que el uso de IA esté declarado;
- que cada persona pueda defender lo que entrega.

Importante:

```text
Si una entrega parece buena pero no puedes explicarla, la evidencia no está completa.
```

## 7. Cómo usar las plantillas

Tendrás plantillas para no empezar cada documento desde cero.

Estarán en:

```text
plantillas/
```

Y también explicadas en:

```text
07-plantillas-entregables.md
```

Algunas plantillas importantes serán:

| Plantilla | Para qué sirve |
|---|---|
| `README-template.md` | Explicar qué hace el proyecto y cómo se ejecuta. |
| `portfolio-template.md` | Explicar qué has hecho y qué has aprendido. |
| `registro-ia-template.md` | Registrar el uso de IA. |
| `pruebas-template.md` | Documentar pruebas. |
| `depuracion-template.md` | Explicar cómo se ha investigado un error. |
| `incidencia-template.md` | Registrar un problema detectado. |
| `retrospectiva-template.md` | Revisar el trabajo del equipo. |
| `defensa-individual-template.md` | Preparar la defensa. |

No todos los hitos usarán todas las plantillas.

En cada hito se indicará cuáles hacen falta.

## 8. Qué se espera en las defensas

La defensa sirve para comprobar que entiendes lo que has entregado.

Puede ser individual, de equipo o una revisión técnica breve.

Se te puede pedir:

- ejecutar el proyecto;
- explicar una función;
- explicar una clase;
- justificar una decisión;
- explicar un error corregido;
- interpretar una prueba;
- modificar una parte pequeña;
- explicar qué parte hizo la IA;
- explicar qué parte hiciste tú;
- comparar Java y Python;
- identificar qué datos no se deben enviar a una IA.

No se trata de memorizar todo.

Se trata de demostrar comprensión real.

Frase clave del curso:

```text
Sé qué he construido, sé cómo funciona, sé cómo lo he comprobado y puedo mejorarlo.
```

## 9. Checklist rápido antes de entregar

Antes de entregar, revisa:

```text
[ ] El proyecto se puede abrir o ejecutar.
[ ] El README explica cómo funciona.
[ ] El código corresponde al nivel del hito.
[ ] Hay pruebas o comprobaciones si se piden.
[ ] He documentado errores o decisiones importantes.
[ ] He registrado el uso de IA si la he usado.
[ ] No he incluido datos personales ni secretos.
[ ] Sé qué parte puedo defender.
[ ] La entrega está subida donde corresponde.
```

## 10. Mensaje final

Este proyecto no va solo de programar.

Va de aprender a trabajar como desarrollador/a en formación:

- pensar;
- construir;
- probar;
- equivocarse;
- depurar;
- documentar;
- colaborar;
- usar IA con criterio;
- defender técnicamente lo aprendido.

El objetivo no es que el agente sea perfecto.

El objetivo es que cada hito demuestre aprendizaje real.
